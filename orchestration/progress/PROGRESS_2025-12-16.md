## 7. Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| **Qwen3-Coder-30B-A3B-Instruct as coder_primary** | 45.3 t/s (4.7x faster than dense); equivalent quality | Qwen2.5-Coder-32B-Instruct (too slow at 9.7 t/s) |
| **Qwen3-Coder-53B-A3B as coder_escalation** | Generalist support for primary failures; 30.4 t/s | Use as primary (slower, no benefit) |
| **Deprecate Qwen2.5-Coder-32B-Instruct** | 16% spec decode acceptance, 4.7x slower than MoE | Keep for compatibility (not worth memory) |
| MoE expert reduction over spec decode for MoE models | Spec decode has low acceptance on MoE (8.96%) | Try other draft models |
| 4 experts for coder roles | Best balance of speed and quality | 2 experts (untested quality) |
| **Use llama-completion/speculative/lookup (not llama-cli)** | llama-cli auto-enables interactive mode for Instruct models, causing hangs | Add --no-conversation flag (unreliable) |
| **env OMP_NUM_THREADS=1 after timeout** | timeout runs in subshell, env vars must be passed via `env` command | Set env var before timeout (doesn't work) |
